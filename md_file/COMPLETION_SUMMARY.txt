## 🎯 Cấu Trúc Tổng Quát Đề Xuất

### **From (Current)**:
```
Ingest CVE → Extract → Store in Graph → Query/Retrieve
(Static knowledge base system)
```

### **To (Proposed)**:
```
┌─────────────────────────────────────┐
│  Define Attack Target               │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│  LLM Decides Action                 │
│  ├─ Query Graph: Current state?     │
│  └─ Query Vector: What exploit?     │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│  Execute Tool (Nmap/Nikto/MSF)      │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│  Parse Output (NEW!)                │
│  └─ Extract entities & relationships│
└─────────────────────────────────────┘
         ↓
    ┌──────────────┬──────────────┐
    ↓              ↓              ↓
  [Graph]   [Vector]   [Session]
  (state)  (techniques) (memory)
    ↓              ↓              ↓
    └──────────────┴──────────────┘
         ↓
┌─────────────────────────────────────┐
│  Goal Achieved? → Continue or End   │
└─────────────────────────────────────┘
```

## 🔑 Key Differences

| Aspect | Current | Proposed |
|--------|---------|----------|
| **Flow** | Linear (one-pass) | Cyclic (iterative) |
| **Inputs** | Static documents | Dynamic tool outputs |
| **State Management** | Workflow state | Graph database (persistent) |
| **Vector DB Use** | CVE chunks | Attack techniques/payloads |
| **Graph DB Use** | CVE relationships | Attack state + CVE knowledge |
| **Parsing** | Document parsing | **Log parsing (NEW)** |
| **Decision Making** | DAG workflow | **LLM with graph/vector queries** |
| **Execution** | Information retrieval | **Tool execution + loop** |

## 📋 The 5 Major Changes

### 1. **Add Log Parser Module** (CRITICAL)
- Parse Nmap JSON → Extract IP, Port, Service
- Parse Nikto XML → Extract vulnerabilities
- Parse Metasploit JSON → Extract exploits
- Generic regex parser for other tools

### 2. **Refactor Neo4j Schema**
- From: CVE/CWE knowledge graph
- To: Attack state graph (IP:Port:Service:Vulnerability)
- Keep: Existing CVE knowledge (for reference)
- Add: New Cypher queries for LLM decisions

### 3. **Shift Vector DB Content**
- From: CVE document chunks
- To: Attack techniques & payloads
- Source: HackTricks, Exploit-DB, Metasploit modules
- Purpose: Semantic search for "What exploit for this vuln?"

### 4. **Redesign LangGraph Workflow**
- From: Linear DAG (planner → retrieval → reasoning → tool → report)
- To: Cyclic loop (decide → execute → parse → loop or end)
- Add: Feedback mechanism (tool output → graph update → new decision)

### 5. **Expand Tool Execution**
- From: Stub implementations
- To: Real tool execution (Nmap, Nikto, Metasploit)
- Sandboxed: Secure execution environment
- Parsed: Convert outputs to structured entities
